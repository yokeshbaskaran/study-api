What is Postman?
---
-> Postman is a tool that helps you create, test, and
use APIs easily.
-> It makes every step of working with APIs simple -
from building them to trying them out.
-> It also helps teams collaborate better when
working on APIs.
-> That's why more than 25 million people use
Postman around the world!

 
Working with APIs: Then vs NOW:
---
 1. Before Postman: Developers used a command-line
tool called "cURL" command in cmd to test APIs
 2. Now with Postman: You get an easy-to-use
interface to test APIs without typing long
commands

-> Postman also makes it easier to share and
work with your team

What is a Collection in Postman?
-------------------------------------
- A collection in Postman is like a folder where you keep your saved API requests.
- Think of it like this:
    Collection = A folder
- API Requests = The files inside it

Postman Conceps:
----------------
1. Collections = A folder
2. API Request Methods
3. Request Response pattern
4. Set a variable
{{bookURL}}
bookURL = https://library-api.postmanlabs.com

5. Query parameter -> ?, &, =
eg:- {{bookURL}}/books?genre=fiction&yearPublished=1960

6. filter using Query parameter
eg:- {{bookURL}}/books?genre=fiction&yearPublished=1960

7. mutliple Query parameter
8. path parameter

-----

Diff Path Parameter and Query Parameter?
-----
Path Parameter	                Query Parameter
--------------------------------------------------
Identifies specific resource	Filters/modifies data
Required usually	            Optional usually
Part of URL path	            Comes after ?
/users/10	                    /users?age=20

What are Headers in API Request?

- Headers are like the outside of an envelope when you
send a letter.
- They carry extra information (metadata) about your
request, such as:

. Who is sending the request (% Authorization)
· What kind of data is expected in return (
Content-Type)
. Extra settings the server needs to know

- But headers don't contain the actual data - that's
what the body is for (like the letter inside the
envelope)

        Header = Delivery Info
        Body = The Real Message/Data
- This helps the server understand how to handle your
request properly!
